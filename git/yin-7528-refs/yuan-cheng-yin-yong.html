远程引用

远程引用

是执行推送操作时记录当前分支所指向的提交对象，并以文件的形式保存：refs/remotes/远程仓库名称/远程仓库的分支名

只有发生过推送操作，才存在远程引用

如：当前分支一直未执行过远程推送操作在refs目录下就没有远程引用文件

ls .git/refs/remotes/origin/

HEAD

执行push操作创建一个远程引用

git push -f origin

ls .git/refs/remotes/origin/
HEAD   master

cat .git/refs/remotes/origin/master
cd3e5c0f5770f1c8d205b9fd9d905a4ff8c616c3

git log
commit cd3e5c0f5770f1c8d205b9fd9d905a4ff8c616c3 (HEAD -> master, origin/master, origin/HEAD)

远程引用文件中保存是当前分支最后一次的提交对象

远程引用文件名是远程仓库分支的名称

如：将分支aaa push到远程分支master 上

git checkout aaa

git push origin master

ls .git/refs/remotes/origin/
HEAD   master

远程分支到本地分支映射方式

引用规格:定义了远程仓库的url地址，远程版本库中的引用以及本地存储这些引用的位置

  • fetch(获取)

    • 默认远程引用 如：remote ：bh代表远程仓库的别名 url:地址 fetch=拉取远程版本库的引用:本地存储引用的位置
    • [remote "bh"]
    url = https://github.com/baihui/spring-boot.git
    fetch = +refs/heads/*:refs/remotes/bh/*

      git remote add** 会自动生成引用规格，但是在默认情况下该命令会获取服务器中 “ .git/refs/heads/“ 下面所有引用同时写入 到默认的位置（efs/remotes/origin/）,当然也可以查看远程的提交日志如：git log bh/tex

    • 自定义远程引用修改.git/config文件 可以自定义指定的远程引用，无需引用所有的，如：在进行 fetch 操作时只更新指定的text分支

      [remote "bh"]

    url = https://github.com/baihui/spring-boot.git
    fetch = +refs/heads/text:refs/remotes/bh/text

  • push(推送)

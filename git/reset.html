reset

reset命令是将HEAD代表的分支指向的提交id修改成指定的提交id，[--soft | --mixed | --hard | --merge | --keep ] [-q] 传递不同参数会涉及到被断开commit数据是否保留到暂存区或者丢弃

如:


git log --oneline

f6a3d03a41 (HEAD -> 4.0.0-RC2-release, origin/dev) fix g4 for assignmentValueNode get error by index (#3290)
247c9182ce Add sharding + encrypt test engine (#3298)
bd885b8cbe Add table owner for logic encrypt column (#3276)
763b02ff56 Add OrderByContextEngineTest (#3293)


#当前HEAD 指向的f6a3d03a41通过 reset 命令可以将他设置指向其中一个

get reset 763b02ff56 #将当前HEAD指向给提交

reset 是修改HEAD分支指向commitID，被替换掉commit数据存在以及id不变，在git log下不显示是因为没有提交或者HEAD引用,.git目录由记录 cat .git/logs/HEAD 如：

git log --oneline 
2a3dc84 (HEAD -> master) ignore
3dc1d42 rebase


git reset --soft HEAD^ 
git commit -a -m "reset 1 commit"

git reset --soft HEAD^ 
git commit -a -m "reset 2 commit"


git reset --soft HEAD^ 
git commit -a -m "reset 3 commit"


#执行后 reset 3 commit 提交显示，被断开提交1 和2 未显示
git log --oneline 
46ee18b (HEAD -> master) reset 3 commit
3dc1d42 rebase

cat .git/logs/HEAD 记录全部过程
#---父亲提交ID-当前提交ID--

git reset --soft ed3e6f415b1a4ee4d8c4518f00429c8fc60c4510

git log --oneline 
ed3e6f4 (HEAD -> master) reset 2 commit
3dc1d42 rebase

git reset 常用格式如下

git reset [--soft | --mixed | --hard | --merge | --keep ] [-q] [commit]
通过reset重新设置HEAD的提交指针时 ，添加不同参数 会影响被断开提交commitId 数据 是否到本地仓库、暂存区或者工作区 以及丢弃
  • --hard 修改本地仓库、暂存区、工作区里面的数据为指定commitId对应快照的内数据
  • --mixed commitId 修改本地仓库、暂存区里面的数据为commitId对应快照里的数据，是git reset默认的参数，--mixed可缺省。 暂存区的数据会被快照中的数据覆盖
  • --soft commitId 修改本地仓库里面的数据为commitId对应快照的数据。(仅改变指向快照的指针指向) 修改过得被被存放到暂存区保留

例如： HEAD^ 将当前HEAD指向提交ID的父提交ID，--soft 当前提交ID对应修改过的文件被暂存区保留

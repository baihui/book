Optional优雅的使用null

使用Optional除了简化粗鲁的if(null == object)、降低函数的复杂度、增加可读性之外，它是一种傻瓜式的防护，Optional引导编码人员主动的思考引用为null的情况，

允许接受可能为null的T类型的引用

//T 是null，
Optional.fromNullable(T)
//若T引用为null则快速失败,java.lang.NullPointerException
Optional.of(T)
//创建引用NULl的Optional实例 
Optional.absent()



Optional<Integer> possible = Optional.fromNullable(5);  //创建允许null值的Optional
possible.isPresent();
possible.get();         // 5
possible.or(3));            // 5
possible.orNull();      // 5
possible.asSet();       // [5]


//....useCache=? 简化粗鲁的if(null == object)
possible = useCache ? Optional.fromNullable(new SQLStatement()) : Optional.<SQLStatement>absent();
if (possible.isPresent()) {
    return possible.get();
}
  • boolean isPresent() 包含的引用非null的（引用存在），返回true
  • T get() 包含的引用缺失，则抛出java.lang.IllegalStateException
  • T or(T) 包含的引用缺失，返回指定的值
  • T orNull 包含的引用缺失，返回null
  • Set asSet() 如果引用存在，返回只有单一元素的集合；引用缺失，返回空集合。
  • Optional transform() 引用存在，则它被转换给定的功能;否则absent()被返回。
  • Iterable presentInstances(Iterable<? extends Optional<? extends T>> optionals) 将参数optionals中为null的optional剔除，同时返回值对象的Iterable

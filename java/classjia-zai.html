Class

class文件是一个二进制文件，以8位字节为单位进行存放，采用C语言中的结构体表示，通过无符号数（u_1...u_8）和表(XXX_info)来表示结构体中不同类型

  • 无符号数 可用来定义 数字，索引引用，数量值，字符串(utf_8)

  • 表 是由多个无符号数或者其他表构成如 方法表，属性表，字段表

同时它也是非常特殊的class对象，它存放在方法区中通过反射可以获取或者调用class对象中信息，而普通Object 对象存放在堆上的， 什么情况下会引起Class对象的加载了显示通过ClassLaoder 其次 调用类的方法（静态属性 或者方法）但是被final修改的不会引发类的初始化?

类的加载过程

第一阶段：加载 加载类处于类装载的第一个阶段。在加载类时，Java虚拟机必须完成以下工作：

  • 通过类的全名，获取类的二进制数据流
    • 通过Class的全限定名来获取二进制流。虚拟机未规定必须重Class 文件中获取，所以可以 网络（url）或者运行时计算生成(Proxy)或者jar
  • 解析类的二进制数据流转换为方法区内的数据结构
    • 解析类的二进制数据流为方法区内的数据结构
  • 创建java.lang.Class类的实例，表示该类型
    • 在内存中生成一个代表这个类的java.lang.Class对象， 作为方法区该类的各种数据的访问入口，java.lang.Class实例是访问类型元数据的接口，也是实现反射的关键数据。

第二阶段：验证 这一阶段的目的是为了确Class文件的字节流包含的信息符合当前虚拟机的要求，并且不会危害虚拟机自身的安全：

  • 文件格式验证：验证字节流是否符合Class文件格式的规范
  • 是否以魔数0xCAFEBABE开头。
  • 版本号：是否 在当 前 虚拟 机 处理 范围 之内。高版本可以执行低版本
  • 常量池的常量是否有不被支持的常量类型
  • 元数据验证：主要目的是对类的元数据信息进行语义校验， 保证不存在不符合 Java 语言规范的元数据信息这个阶段可能包括的验证点如下： 类是否有父类是否继承了不允许被继承的类（被final修饰的类）等。
  • 字节码验证
    1. 保证跳转指令不会跳转到方法体以外的字节码指令上。 保证方法体中的类型转换是有效的
    2. 符号引用验证符号引用中通过字符串描述的全限定名是否能找到对应 的类。在指定类中是否存在符合方法的字段描述符以及简单名称所描述的方法和 字段。

第三阶段：准备 这个阶段主要是对类的变量分配内存空间以及设置默认初始值，这个赋值属于Java虚拟机的行为属于类变量的初始化阶段。在准备阶段，不会有任何Java代码被执行

  • 类的常量字段（被static修饰）
    • static
      • 会被初始化为零或者null ，在类初始化阶段进行真正赋值,生成的函数中，整合了类中的static赋值语句以及static语句块 字节码如下： static {};
    • static final
      • 就会被初始化为真正值（final不可以修改为了性能，所以不会引起该类的初始化）

//那变量value在准备阶段过后的初始值为0而不是123
public static int value= 123；
//变量fvalue在准备阶段就会被初始化为123 所以调用不会引起类初始
public static final int fvalue= 123；

第四阶段：解析阶段

解析阶段是虚拟机将常量池内的部分符号引用替换为直接引用的过程,但是不涉及到类实例创建也就是部分符号引用还无法确定地址无法解析，所以分为两个阶段解析

符号引用 以一组符号来描述所引用的目标，符号可以是任何形式的字面量，只要使用时能无歧义地定位到目标即可。符号引用与虚拟机实现的内存布局无关，引用的目标并不一定已经加载到内存中。在编译阶段将代码调用的目标对象转化为符号引用，这些符号是存放在常量池中 直接引用 可以是直接指向目标的指针、相对偏移量或是一个能间接定位到目标的句柄。直接引用是和虚拟机实现的内存布局相关的，同一个符号引用在不同虚拟机实例上翻译出来的直接引用一般不会相同。如果有了直接引用，那引用的目标必定已经在堆中存在

  • 静态解析 能够解析条件是在该方法区中的符号引用表示的类以完成初始化阶并且有一个可确定的调用版本，所以通过静态解析将符号引用替换成指向内存中的某个偏移地址也就是直接引用

  • 动态解析 无法确定一个可调用版本，如类的实例

五阶段：类初始化

类的初始化是类装载的最后一个阶段此时类才会开始执行Java字节码。初始化阶段的重要工作是执行类的初始化方法该方法是由编译器自动生成且线程安全，它是由类静态成员的赋值语句以及static语句块合并产生的 如



    public static int sbnInt = 88 ;
    public static int sbnIntVoid ;
    static {
        sbnIntVoid = 23 ;

Java虚拟机规定一个类或接口在初次使用前，必须要进行初始化分为如下情况

  • 只有下列几种情况会引发类的初始化
    • new关键字，或者通过反射、克隆、反序列化
    • 使用了字节码invokestatic指令A.xxx()
    • 当使用类或接口的静态字段时（final常量除外），比如，使用getstatic或者putstatic指令A.a=1
    • 当初始化子类时，要求先初始化父类
    • 作为启动虚拟机，含有main()方法的那个类（静态方法）
  • 不会引发类的初始化 在调用一个类字段时，只有直接定义该字段的类才会被初始化
    • 如继承 调用父类的方法或者属性不会引发该类的初始化

LockSupport

全部都是静态方法 所以不能通过构造函数创建一个LockSupport，相应中断但是不会吞掉标志位不抛出异常 是一个可以提前发放通行准许（unpark 不管提前发放多少次，只用于一次性使用），等有阻塞于park 时直接用行，相应中断，但是不抛出异常 也不会吞掉中断标志位，会立刻从park方法返回

  • park（Object o）或者 setBlocker（Object o） 可以通过此方法来 设置 阻塞线程和唤醒线程的数据交换
  • getBlocker（）是返回交换的数据
  • parkNanos (o)阻塞于某个对象上 可以通过唤醒来使其返回。
  • parkNanos(o,l) 等待l时间返回
  • park() 阻塞于当前方法中 ,不会吞掉interrupt标志位 而sleep 会 public static native void sleep(long millis) throws InterruptedException;
  • unpark(T)唤醒被阻塞的线程

把LockSupport视为一个sleep()来用，只是sleep()是定时唤醒，LockSupport既可以定时唤醒，也可以由其它线程唤醒 同时sleep中断会抛异常，但是它不会。

  • String 在代码中没有显示的创建String对象那么当前变量指向的地址是常量池地址也就是说对于字符量预值的变量指向是常量池地址，而现实通过new 指令指向的是堆地址，堆上地址在指向常量池

//添加到常量池
String a = "abc" ;
//添加到常量池
String ab = new String("123") ;
//StringBuilder 加 添加到常量池
String abc = "a"+ab+"123" ;

String y = new Integer(1).toString()+new Integer(2).toString()+new Integer(3).toString();//指向堆
String xy = new String("1")+new String("2")+new String("3") ;////指向堆

x == y  是flase ；因为x中存放的指向字符串池123的索引值，而y中存放的y指向StringBuilder的地址，

x == y.toString.intern() 是true 因为 y指向是通过StringBuilder.append(“1”)把字符串先加在一起的变成一个String 123 而intern() 返回的是123 索引值 所以就相等了，

Intern() 返回字符的常量池地址

String b1 = "a"+"b”; 和 “ab” ;等价
所以 b1 == “ab” 因为指向同一地址

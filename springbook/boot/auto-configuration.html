自动配置

@EnableAutoConfiguration 自动配置标记-在解析@Configuration配置类时候 通过@Import指定ImportBeanDefinitionRegistrar 和 DeferredImportSelector实现类加载在spring.factories文件中预定义配置类以及回调导入选择时发布AutoConfigurationImportEvent事件

完成自动配置条件：通过SpringBootCondition 存在分析在文件中预定配置类的类名称.ConditionalOnClass类是否存在于路径下


@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@Import(AutoConfigurationPackages.Registrar.class)
public @interface AutoConfigurationPackage {

}

@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@AutoConfigurationPackage
@Import(AutoConfigurationImportSelector.class)
public @interface EnableAutoConfiguration {

    String ENABLED_OVERRIDE_PROPERTY = "spring.boot.enableautoconfiguration";

    Class<?>[] exclude() default {};

    String[] excludeName() default {};

}
  • ImportBeanDefinitionRegistrar @Import(AutoConfigurationPackages.Registrar.class) 注册接口

  • DeferredImportSelector @Import(AutoConfigurationImportSelector.class) 是一个延迟选择导入实现着

DeferredImportSelectorGroupingHandler 分组延迟处理器内部类

加载所有org.springframework.boot.autoconfigure.EnableAutoConfiguration名称的配置类值在META-INF/spring.factories文件中

AutoConfigurationMetadataLoader 读取META-INF/spring-autoconfigure-metadata.properties文件定义的配置类依赖的 元数据AutoConfigurationMetadata

加载所有在META-INF/spring.factories 文件中预定义的 AutoConfigurationImportFilter导入拦截器，迭代所有配置类 回调 AutoConfigurationImportFiltermatch 查找配置类名称.ConditionalOnClass 在AutoConfigurationMetadata中是否存在，如果存在尝试在当前类路径下加载该 ***.ConditionalOnClass 如果加载成功进行配置，

这个注释根据pom中添加的jar依赖关系来“猜测”我们如何配置Spring,例如:参加一个web项目需要依赖jar


<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
</dependencies>

由于spring-boot-starter-web添加了Tomcat和Spring MVC


@RestController
@EnableAutoConfiguration
public class Example {

    @RequestMapping("/")
    String home() {
        return "Hello World!";
    }

    public static void main(String[] args) {
        SpringApplication.run(Example.class, args);
    }

}

SpringApplication通过调用委托给run方法的类来引导我们的应用程序启动Spring，以及启动自动配置的Tomcat Web服务器

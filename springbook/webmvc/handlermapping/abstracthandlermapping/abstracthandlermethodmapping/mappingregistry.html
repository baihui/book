MappingRegistry

是AbstractHandlerMethodMapping的内部类用于封装映射关系

//url 和 URL 包含映射
private final MultiValueMap<String, T> urlLookup = new LinkedMultiValueMap<>()

//url 和 HandlerMethod
private final Map<T, HandlerMethod> mappingLookup = new LinkedHashMap<>();
//一个策略名称对应多个HandlerMethod
private final Map<String, List<HandlerMethod>> nameLookup = new ConcurrentHashMap<>();

如：注册url和HandlerMethod(handler和Method)映射关系

@Controller
class MyHandler {
@RequestMapping     
public void handlerMethod1() {
}
@RequestMapping
public void handlerMethod2() {
}
}

String url = "/tex" ;
MyHandler handler = new MyHandler();
Method method = handler.getClass().getMethod("handlerMethod1");

MappingRegistry mappingRegistry = new MappingRegistry()
mappingRegistry.registerMapping(url, handler, method);
  • mappingRegistry.registerMapping(url, handler, method) 注册流程
    • 封装handler和method为HandlerMethod(handler, method)注册到mappingLookup中
    • 验证url是否已经在mappingLookup中存在，抛异常
    • 注册HandlerMethod到mappingLookup中
    • 检查当前url是否还包含其他directUrl，如果存在注册到urlLookup中
    • 返回策略名称HandlerMethodMappingNamingStrategy
    • 策略名称与HandlerMethod注册到nameLookup中(一对多关系)

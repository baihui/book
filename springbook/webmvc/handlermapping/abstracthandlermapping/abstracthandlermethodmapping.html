AbstractHandlerMethodMapping

是AbstractHandlerMapping实现的抽象子类，实现了注册映射，返回给定请求的执行链以及取消注册给定的映射等。

由一个内部类MappingRegistry保存mapping和HandlerMethod之间的映射关系

对于每个已注册的处理程序方法，维护Method和Handler唯一映射，其中子类定义映射类型

抽象模板方法



 //给定类型是否是具有处理程序方法的处理程序
protected abstract boolean isHandler(Class<?> beanType);

//返回处理程序方法的映射。
protected abstract T getMappingForMethod(Method method, Class<?> handlerType);

//提取并返回映射mapping中包含的URL路径。如：/** 
protected abstract Set<String> getMappingPathPatterns(T mapping);

//检查映射是否与当前请求匹配，并返回具有与当前请求相关的条件的（可能是新的）映射。
protected abstract T getMatchingMapping(T mapping, HttpServletRequest request);

//返回比较器以对匹配映射进行排序。返回的比较器应该排序'更好'匹配更高。
protected abstract Comparator<T> getMappingComparator(HttpServletRequest request);

注册给定的映射

public void registerMapping(T mapping, Object handler, Method method) {
    this.mappingRegistry.register(mapping, handler, method);
}

给定请求(request)获取处理执行链HandlerExecutionChain执行步骤如下：

  • 查找给定请求(request)的处理程序方法HandlerMethod
protected HandlerMethod getHandlerInternal(HttpServletRequest request) {
    //解析出路径
    String lookupPath = getUrlPathHelper().getLookupPathForRequest(request);
    this.mappingRegistry.acquireReadLock();
    try {
            //从MappingRegistry操作路径匹配HandlerMethod 
        HandlerMethod handlerMethod = lookupHandlerMethod(lookupPath, request);
        //如果提供的实例包含bean名称,解析成Bean
        return (handlerMethod != null ? handlerMethod.createWithResolvedBean() : null);
    }
    finally {
        this.mappingRegistry.releaseReadLock();
    }
}

在注册Mapping时handel提供的是Bean名称而不是实例，在返回HandlerMethod会解析成Bean

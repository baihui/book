镜像

Docker镜像由一系列层组成。每层代表镜像文件Dockerfile中的一条指令。除最后一层外，每一层都是只读的,如果对其写操作 通过写入时复制（CoW）策略到写入层中


FROM ubuntu:18.04
COPY . /app
RUN make /app
CMD python /app/app.py

该Dockerfile包含四个命令，每个命令创建一个层。

  • FROM语句首先从ubuntu:18.04创建层。
  • COPY命令从Docker客户端的当前目录添加一些文件。
  • RUN命令使用命令构建应用程序make。
  • CMD 最后一层指定在容器中运行什么命令

每一层只是与上一层不同的一组。这些层彼此堆叠,创建新容器时，可以在基础层之上添加新的可写层。该层通常称为 容器层 ,对运行中的容器所做的所有更改（例如写入新文件，修改现有文件和删除文件）都将写入可写容器层

容器和镜像

容器和镜像之间的主要区别是可写顶层。在容器中添加新数据或修改现有数据的所有写操作都存储在此可写层中。删除容器后，可写层也会被删除。基础镜像保持不变，每个容器都有其自己的可写容器层，并且所有更改都存储在该容器层中，

  • 多个容器可以共享对同一基础镜像的访问，但具有自己的数据状态.
  • 镜像之间可以通过卷来共享数据
当通过 docker pull拉取或者Dockerfile build 镜像时，每个层都会被分别下拉，并存储在Docker的本地存储区域中(/var/lib/docker/),如

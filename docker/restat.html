重启策略

Docker提供了重启策略来控制容器在退出时以及在Docker重新启动时自动启动容器

docker run 或者dokcer update 使用重启策略 --restart

docker服务启动时针对指定容器服务重启策略，分别如下几种：

  • no 容器退出时，不重启容器
  • on-failure 在容器非正常退出时（退出状态非0），才会重启容器
  • always 无论退出状态是如何，都重启容器
  • unless-stopped 与相似always，除了在容器停止（手动或其他方式）时，即使重新启动Docker守护程序也不会重新启动容器。

    on-failure和always 区别在于on-failure是依据容器退出状态，决定是否重启容器并且还可以指定启动失败的次数 ，而always

例如


#当前docker服务运行两个容器
docker ps -a 

CONTAINER ID        IMAGE              
e90b629545c2        nextcloud           
cda804d99ab5        mysql:5.7.20

#修改nextcloud容器在不正常退出的重启并且失败启动3次
docker update nextcloud --restart on-failure:3


#启动Redis容器并将其配置为始终重新启动，除非明确将其停止或重新启动Docker
docker run -dit --restart unless-stopped redis

mysql

sudo docker run -d -p 3306:3306 -v /home/baihui/mysqls/3306/conf/:/etc/mysql/mysql.conf.d/ -v /home/baihui/mysqls/3306/mysql:/var/lib/mysql 0e56d527a63e -v /home/baihui/mysqls/sql:/home -e MYSQL_ROOT_PASSWORD=root  --restart=always mysql:5.7.20

使用重新启动策略时以下几点：

  • 重新启动策略仅在容器成功启动后才生效。在这种情况下，成功启动意味着该容器已启动至少10秒钟，并且Docker已开始对其进行监视。这样可以防止根本无法启动的容器进入重启循环。
  • 如果手动停止容器，则其重新启动策略将被忽略，直到Docker守护进程重新启动或手动重新启动容器为止。这是防止重新启动循环的另一种尝试。
  • 重新启动策略仅适用于容器。群服务的重启策略配置不同.
使用流程管理器(外部的进程依赖Docker容器时) upstart， systemd或 supervisor 脚步启动

#!/bin/bash
# init file
#启动容器脚步
echo "启动"
docker start bceefaeb0c3e

请勿尝试将Docker重新启动策略与主机级流程管理器结合使用，因为这会产生冲突

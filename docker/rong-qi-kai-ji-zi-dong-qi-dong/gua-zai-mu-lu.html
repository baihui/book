挂载目录

-v

注意权限文件，有时候容器中执行的程序可以不是以root 执行，这样导致宿主的创建目前权限 容器程序无法访问

如：在挂载mysql的日志目录

sudo docker run -d -p 3306:3306 -v /home/baihui/mysqls/3306/conf/:/etc/mysql/mysql.conf.d/ 
-v /home/baihui/mysqls/3306/mysql:/var/lib/mysql 
-v /home/baihui/mysqls/3306/log/:/var/log/mysql/  
-v /home/baihui/mysqls/sql:/home -e MYSQL_ROOT_PASSWORD=root --restart=always mysql:5.7.20


#/var/log/mysql/  在容器中是以mysql用户，通过-v挂载宿主目录/home/baihui/mysqls/3306/log/ 是当前用户创建的，导致无法读写
#修改 调整宿主目录log权限 ok了
